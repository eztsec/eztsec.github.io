This target machine tests the combined exploitation of vulnerabilities, including SQL injection, privilege escalation, file inclusion, and file upload. The exploitation of these vulnerabilities together requires flexible and broad thinking.

20240820 holynix

Host Discovery

​​

Port Scanning

​​

​​

​​

Vulnerability Discovery

Upon accessing port 80, a login page is discovered.

​​
SQL injection vulnerability is found in the password input field.

​​

Vulnerability Exploitation

SQL Injection

At this point, data can be dumped via SQL injection, but a universal password test can be performed first.

Successful login.

​​

File Upload

​​

The current user does not support file upload, and it is known that the current user is ‘alamo’. An attempt is made to switch to another user.

There are two methods here: privilege escalation or using SQL injection to construct a new user login.

Method One: Horizontal Privilege Escalation

Change the uid parameter in the cookie of the HTTP request header and resend the request package.

​​

​​

Method Two: SQL Injection for User Construction

Construct a new user login with SQL injection (or directly dump data via SQL injection during login to log in with the revealed password).

​​

File Upload

​​

​​

Access the directory /~etenenbaum/shell.php, but it is found that the file cannot be executed. (According to the directory scanning rules, to access a user’s home directory, a tilde ~ can be added before the username.)

​​

File Inclusion

There is a file inclusion vulnerability at http://192.168.119.142/index.php?page=ssp.php.

​​

Attempt to exploit this vulnerability to read the file upload code source.

The source code for file upload processing is displayed after the file is uploaded.

​​

Read the transfer.php file.

​​

From the code file, it is known that only tar.gz files uploaded can be unpacked and moved to the directory for execution, so a tar.gz package for file upload shell is prepared.

Initial Access

Successfully executed

http://192.168.119.142/~etenenbaum/shell3.php

​​

Privilege Escalation

Using sudo -l to check, it is found that the tar and mv commands in the /bin directory can be executed with root privileges without a password.

​​

Thus, the mv command can be used to replace the sh command with tar, allowing the use of the root shell without a password.
​​
