Access WordPress

Since the response package needs to load the js file from 192.168.159.145, but the system’s IP address is not 192.168.159.145, it needs to be changed to 192.168.48.135. Therefore, set up a replacement for the IP in the response package in Burp.

Known Vulnerable Malicious Plugin

This method involves installing a malicious plugin that is known to have vulnerabilities and can be exploited to obtain a web shell. This process is carried out through the WordPress dashboard as follows:

  1. Plugin Acquisition：Obtain plugins from sources such as Exploit DB, such as **here**。
  2. Plugin Installation：
  • Navigate to the WordPress dashboard, then go to Dashboard > Plugins > Upload Plugin.​
  • Upload the downloaded plugin zip file.
  1. Plugin Activation：After the plugin is successfully installed, it must be activated through the dashboard.
  2. Exploitation：
  • After installing and activating the “reflex-gallery” plugin, you can exploit it because it is known to have vulnerabilities.

Privilege Escalation

Check the wp-config.php file and find the database password

​​

Since the SSH port is not open, this password cannot be used to try an SSH connection.

Check the /etc/passwd file to see the login users

​​

wpadmin can log in, try to use the adam14 password to escalate privileges to this user, and log in successfully

​​

Check the sudo permissions and find that mysql uses root permissions to execute /usr/bin/mysql -u root -D wordpress -p

​​

So log in to the database, enter mysql and use the system to execute root-level commands

​​

Privilege escalation successful
